# 条目模板

每条新记录追加到 `$AI_LOOPS_DATA/ai-deviation-log.md` 末尾,使用以下格式:

```markdown
## #<编号> | <YYYY-MM-DD HH:MM> | <一句话标题>

**场景**: <可选,在做什么时遇到的>

**AI 给的方案**:
<原方案,粘贴或概括>

**错在哪**:
<错误点>

**我怎么发现的**:
<触发警觉的信号 — 最重要的一栏>

**失效归类**: `<category>`

**修正后的方案** (可选):
<正确做法,可省略>

---
```

## 字段填写要点

- **一句话标题**:能让自己 3 个月后扫一眼就回忆起场景,例如 "AI 给的 Redis 锁释放方案漏掉了 token 校验"
- **AI 给的方案**:**粘贴关键片段**比抽象描述好;太长就抽要点 3-5 行
- **错在哪**:精确指出错点,避免 "不太对" 这种模糊表达
- **我怎么发现的**:必填,最值钱的一栏。例如:
  - "看到 `Set` 后没 `if get == token` 就直觉不对"
  - "AI 给的数字和我心算对不上"
  - "AI 引用了一个我没听过的 API,搜了下不存在"
- **失效归类**:从 SKILL.md 标准类别选;新增类别要在末尾"扩展类别"区登记一条
